# 2026-04-26 批量参数压测结论

测试目标：
- 固定 `mongo_profile="recommended"`
- 固定 `streaming=true`
- 比较 `batch_size` 与 `mongo_batch_size` 对吞吐的影响

测试对象：
- 数据源：`PostGIS`
- 输出：`MongoDB`
- 表：`public.ht_tyg5c32ihg_sys_ht_mark`

## 当前结论

| batch_size | mongo_batch_size | 耗时 | tile 数 |
| ---: | ---: | ---: | ---: |
| `20000` | `8192` | `233.60s` | `20395` |
| `5000` | `8192` | `242.30s` | `20395` |
| `20000` | `4096` | `247.44s` | `20395` |
| `10000` | `8192` | `249.03s` | `20395` |
| `5000` | `2048` | `251.96s` | `20395` |
| `20000` | `2048` | `252.40s` | `20395` |
| `10000` | `4096` | `258.27s` | `20395` |
| `5000` | `4096` | `259.10s` | `20395` |
| `10000` | `2048` | `275.22s` | `20395` |

## 现状判断

- 9 组组合产出的 tile 数完全一致，说明这一轮只影响吞吐，不影响结果正确性。
- `mongo_batch_size=8192` 在三档 `batch_size` 下整体更优，说明更大的 Mongo 批量写入对当前表更有利。
- `batch_size=20000` 拿到了全场最快结果，说明更大的 PostGIS 读取批次对当前大表是正收益。

## 当前默认值

推荐默认吞吐参数：

```text
batch_size = 20000
mongo_batch_size = 8192
```

理由：
- 在真实大表上是最快组合。
- 结果与其它组合一致，没有引入额外 tile 风险。
- 相比旧基线 `10000 / 4096`，提升明确，且没有增加复杂度。

保守备选：

```text
batch_size = 10000
mongo_batch_size = 8192
```

适用场景：
- 如果希望保持较保守的 PostGIS 读取批次，这个组合也优于旧基线。

## 结论

- 当前项目的默认吞吐参数已经调整为 `20000 / 8192`。
- 这组默认值适用于当前 `recommended` profile 的生产主路径。
